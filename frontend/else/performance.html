#性能优化

白屏： FCP - 路由改变时

  1. 回车按下,浏览器解析网址,进行 DNS 查询,查询返回 IP,通过 IP 发出 HTTP(S) 请求
  2. 服务器返回HTML,浏览器开始解析 HTML,此时触发请求 js 和 css 资源
  3. js 被加载,开始执行 js,调用各种函数创建 DOM 并渲染到根节点,直到第一个可见元素产生

网络上的优化：

  • DNS: prefetch
  • CDN
  • 域名分片，HTTP2后作用不大
  • 资源压缩
  • 开启HTTP2：
    • 二进制分帧的方式通信，http1.x 是用文本,http2 的效率更高
    • 多路复用，减少TCP多次连接的时间
    • 头部压缩
    • 服务端推送，加载HTML时就推送js和css文件
  • 浏览器缓存
    • 拆包：抽离基础包，工具包和特定库等。chunk id变化使用HashedModuleIdsPlugin

针对HTML文件：

  1. loading提示：html-webpack-plugin可配置loading
  2. 服务端渲染SSR和Prerendering
  3. FCP 和 FMP之间（数据还没回来前），使用占位图（骨架图）

文件加载的优化：

  • treeShaking

    • 使用库的ESM版本
    • 关闭Babel的module：false

  • polyfill动态加载：<script src="https://cdn.polyfill.io/v2/polyfill.min.js"></script>

  • 动态加载ES6代码

  • 路由级别的懒加载

    • Vue： babel插件plugin-syntax-dynamic-import后使用component: () => import('@components/login')
    • React: React.lazy()

  • 组件的加载

    • 组件懒加载： lazy + Suspense
    // 动态加载图表组件
const Chart = lazy(() => import(/* webpackChunkName: 'chart' */'./charts'))

// 包含着图表的 modal 组件
const ModalEchart = (props) => (
    <Modal
    	onOk={props.handleOk}
  	>
    	<Chart />
  	</Modal>
)
    • 组件预加载： 提前触发组件的加载
    • Vue的keep-alive

    其他：

    • 图片懒加载
    • 雪碧图， HTTP2后作用不大
